Domácí síťová infrastruktura - Fedora server + Windows 11

Úloha: Windows 11 slouží jako správcovský počítač. Fedora Linux je centrální multimediální server. Síť běží ve VMware přes NAT v rozsahu třídy B: 172.16.0.0/16.

1. VMware síť

Klikni: VM Settings → Network Adapter → NAT Nastav to u:
  • Windows 11 VM
  • Fedora Server VM
Použitá síť: 172.16.0.0/16 Gateway: 172.16.0.1 Statická IP Fedory: 172.16.255.254

2. Instalace základních balíčků na Fedoře

Aktualizace: sudo dnf update -y Instalace služeb: sudo dnf install openssh-server cockpit samba httpd tigervnc-server lvm2 nano tar firewalld -y Instalace KDE Plasma: sudo dnf groupinstall "KDE Plasma Workspaces" -y Zapnutí grafického režimu: sudo systemctl set-default graphical.target

3. Statická IP adresa

Zjisti název síťového připojení: nmcli con show Nastavení statické IP: sudo nmcli con mod "Wired connection 1" ipv4.addresses 172.16.255.254/16 sudo nmcli con mod "Wired connection 1" ipv4.gateway 172.16.0.1 sudo nmcli con mod "Wired connection 1" ipv4.dns 8.8.8.8 sudo nmcli con mod "Wired connection 1" ipv4.method manual Restart připojení: sudo nmcli con up "Wired connection 1" Ověření: ip a ping 8.8.8.8

4. SSH přístup pomocí klíčů

Na Fedoře zapni SSH: sudo systemctl enable --now sshd Na Windows 11 otevři PowerShell: ssh-keygen Zkopíruj veřejný klíč na server: scp C:\Users\TVUJ_UZIVATEL\.ssh\id_rsa.pub user@172.16.255.254:/home/user/ Na Fedoře: mkdir -p ~/.ssh cat ~/id_rsa.pub >> ~/.ssh/authorized_keys chmod 700 ~/.ssh chmod 600 ~/.ssh/authorized_keys Test z Windows: ssh user@172.16.255.254

5. VNC vzdálená plocha

Na Fedoře spusť: vncserver Nastav VNC heslo. Na Windows otevři VNC Viewer a připoj se na: 172.16.255.254:5901

6. Firewall - firewalld

Zapnutí firewallu: sudo systemctl enable --now firewalld Povol pouze nutné služby: sudo firewall-cmd --permanent --add-service=ssh sudo firewall-cmd --permanent --add-port=5901/tcp sudo firewall-cmd --permanent --add-port=8080/tcp sudo firewall-cmd --permanent --add-port=8081/tcp sudo firewall-cmd --permanent --add-port=9090/tcp sudo firewall-cmd --permanent --add-service=samba Načtení pravidel: sudo firewall-cmd --reload Ověření: sudo firewall-cmd --list-all

7. Cockpit

Zapnutí Cockpitu: sudo systemctl enable --now cockpit.socket Na Windows otevři v prohlížeči: https://172.16.255.254:9090 Přihlas se účtem z Fedory.

8. Samba sdílení SHARE

Vytvoř složku: mkdir -p /home/user/SHARE chmod 777 /home/user/SHARE Otevři konfiguraci: sudo nano /etc/samba/smb.conf Na konec vlož: 
[SHARE]
path = /home/user/SHARE
browsable = yes
writable = yes
guest ok = yes
read only = no
Zapni Sambu: sudo systemctl enable --now smb nmb Restart: sudo systemctl restart smb nmb Ve Windows otevři Průzkumník souborů: \\172.16.255.254\SHARE

9. Apache weby na portech 8080 a 8081

Vytvoř webové složky: sudo mkdir -p /var/www/web8080 sudo mkdir -p /var/www/web8081 Testovací stránky: echo "WEB 8080 funguje" | sudo tee /var/www/web8080/index.html echo "WEB 8081 funguje" | sudo tee /var/www/web8081/index.html Otevři konfiguraci Apache: sudo nano /etc/httpd/conf/httpd.conf Na konec vlož: 
Listen 8080
Listen 8081

<VirtualHost *:8080>
DocumentRoot "/var/www/web8080"
</VirtualHost>

<VirtualHost *:8081>
DocumentRoot "/var/www/web8081"
</VirtualHost>
Zapni Apache: sudo systemctl enable --now httpd Restart: sudo systemctl restart httpd Ověření z Windows: http://172.16.255.254:8080 http://172.16.255.254:8081

10. SCP / SFTP nahrání index.html

Na Windows vytvoř soubor: index.html Nahraj soubor přes SCP: scp index.html user@172.16.255.254:/home/user/ Na Fedoře překopíruj do obou webů: sudo cp /home/user/index.html /var/www/web8080/index.html sudo cp /home/user/index.html /var/www/web8081/index.html Ověř v prohlížeči: http://172.16.255.254:8080 http://172.16.255.254:8081

11. LVM ze dvou datových disků

Zobraz disky: lsblk Předpoklad: /dev/sdb /dev/sdc Vytvoř fyzické svazky: sudo pvcreate /dev/sdb /dev/sdc Vytvoř skupinu disků: sudo vgcreate data_vg /dev/sdb /dev/sdc Vytvoř logický svazek: sudo lvcreate -l 100%FREE -n data_lv data_vg Naformátuj: sudo mkfs.ext4 /dev/data_vg/data_lv Připoj: sudo mkdir /mnt/data sudo mount /dev/data_vg/data_lv /mnt/data Ověření: lsblk df -h

12. Backup skript

Vytvoř adresář mimo LVM: sudo mkdir /backup sudo chmod 777 /backup Vytvoř skript: nano /home/user/backup.sh Vlož: 
#!/bin/bash
tar -czf /backup/backup_$(date +%F).tar.gz /home/user/SHARE
Nastav práva: chmod +x /home/user/backup.sh Spusť: /home/user/backup.sh Ověř zálohu: ls -lh /backup

13. Automatizace přes cron

Otevři cron: crontab -e Denní záloha ve 2:00: 0 2 * * * /home/user/backup.sh

14. KeePass

Na Windows nainstaluj KeePass. Do databáze ulož:
  • Fedora SSH účet
  • VNC heslo
  • Samba přístup
  • Cockpit přístup

15. Test firewallu

Z Windows otestuj povolený port: Test-NetConnection 172.16.255.254 -Port 22 Otestuj zakázaný port: Test-NetConnection 172.16.255.254 -Port 1234 Port 22 má projít, port 1234 má být blokovaný.
